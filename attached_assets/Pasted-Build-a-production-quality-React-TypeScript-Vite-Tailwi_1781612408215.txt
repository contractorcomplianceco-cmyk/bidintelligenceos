Build a production-quality React + TypeScript + Vite + Tailwind prototype for:

CCA BidIntelligenceOS
Tagline: Research Less, Win More

PRODUCT TYPE
This is an external contractor-facing product that contractors buy and use to win more bids. It is NOT a CCA internal tool, NOT CCA’s internal pricing formula model, and NOT a CCA sales/proposal tool.

ARCHITECTURE FLEXIBILITY
Build the prototype so it can work in either of these future paths:

1. Standalone System:
   - BidIntelligenceOS operates as its own SaaS-style contractor workspace.
   - Contractors log in, manage bid opportunities, upload bid history, generate strategy, build bid packages, and monitor outcomes.

2. ContractorConnect Add-On:
   - BidIntelligenceOS can also be positioned as an add-on module inside ContractorConnect.
   - In this mode, it should feel like a premium “Bid Intelligence” workspace connected to a contractor’s existing ContractorConnect profile, compliance readiness, documents, and contractor growth tools.

For the prototype, include a visible toggle or product context selector:
- Standalone Mode
- ContractorConnect Add-On Mode

The UI should adapt labels slightly based on the selected mode, but the core workflow should remain the same.

NON-NEGOTIABLE BOUNDARIES
- Do not expose or reference CCA’s internal Pricing Formula Model.
- Do not build this as an internal CCA tool.
- Do not promise guaranteed bid wins.
- Do not say the AI can guarantee pricing accuracy, agency outcomes, legal compliance, bonding approval, licensing approval, procurement awards, or bid awards.
- Do not expose internal cost/margin notes in the vendor-facing bid package unless the user intentionally includes them.
- Keep internal bid strategy separate from vendor-facing bid package output.
- All AI output must require user review before export.

Build a premium construction-tech SaaS interface.

- First screen must be the actual contractor dashboard
